# 《Web开发技术》课程设计前端页面设计方案（Agent统一参考版）

> 项目定位：本文件作为后续所有开发、设计、文案、数据库、后端、测试 Agent 的统一参考文档。任何页面设计、接口字段、权限逻辑和前端实现都应优先遵守本文档。

---

## 1. 课程要求提炼

课程设计主题为“中华优秀传统文化”，要求完成一个完整网站开发。前端设计必须满足以下硬性要求：

1. 网站主题自拟，但必须围绕中华优秀传统文化，定位明确，内容积极健康，素材可从网络搜集，但必须标注出处。
2. 页面设计必须具有原创性，主题鲜明，内容充实。
3. 网站结构规划合理，目录结构规范，目录命名使用有意义的非中文名称。
4. 网站至少包含三级页面，子页风格统一，布局美观，色彩协调。
5. 首页必须命名为 `index.html`。
6. 必须实现响应式页面设计，适配桌面端、平板端、移动端。
7. 注册功能需要连接数据库，完成用户信息注册并动态更新数据库。
8. 登录页面需要验证码，并与数据库连接，实现用户信息和验证码验证。
9. 注册用户可以对子网页内容进行点赞、评论、转发。
10. 未登录用户只能以游客身份访问首页，其余子页需要屏蔽，无访问权限。
11. 后台需要查看所有子网页的点赞、评论等数据。

## 2. 网站主题方案

### 2.1 网站名称

**华夏文脉 · 中华优秀传统文化数字展馆**

英文项目名与目录推荐：

```text
heritage-museum
```

### 2.2 核心定位

本网站以“中华优秀传统文化”为主题，采用数字展馆式页面结构，将中华传统文化中的思想理念、精神特质、物质文化、节日民俗、科学技艺、生态智慧等内容组织成可浏览、可互动、可管理的专题网站。网站首页面向所有访客展示整体文化主题，登录后用户可以进入具体子页阅读内容，并对文化内容进行点赞、评论、转发。后台管理员可查看各页面互动数据，用于课程设计中的动态功能展示。

### 2.3 设计关键词

```text
典雅、现代、留白、卷轴、宣纸、朱砂、鎏金、青绿山水、数字展馆、卡片式内容、响应式交互
```

### 2.4 目标用户

| 用户类型 | 访问权限 | 主要需求 |
|---|---|---|
| 游客 | 只能访问首页 | 快速了解网站主题、查看精选文化介绍、引导登录/注册 |
| 注册用户 | 首页 + 所有文化子页 + 互动功能 | 浏览专题、阅读详情、点赞、评论、转发 |
| 管理员 | 前台页面 + 后台管理页 | 查看用户、文章、点赞、评论、转发统计 |

## 4. 页面设计总览

### 4.1 首页 `index.html`

**访问权限：游客与登录用户均可访问。**

首页目标是完成主题展示和登录引导。游客只能访问该页，点击子页卡片时应提示登录。

#### 页面模块

1. 顶部导航栏
   - Logo：华夏文脉
   - 导航：首页、文化专题、数字展馆、关于本站
   - 未登录：登录、注册按钮
   - 已登录：用户头像、用户中心、退出登录
2. Hero 首屏
   - 大标题：华夏文脉 · 中华优秀传统文化数字展馆
   - 副标题：在现代网页中重读传统文化的思想、器物与精神
   - 主按钮：进入展馆
   - 次按钮：注册体验
   - 背景：宣纸纹理 + 青绿山水淡纹 + 朱砂印章元素
3. 文化分类卡片
   - 思想理念
   - 精神品格
   - 器物与非遗
   - 节日民俗
   - 科学技艺
   - 生态智慧
4. 精选内容区
   - 展示 3 到 6 篇推荐文章卡片
   - 卡片含标题、摘要、来源、标签、互动数预览
5. 文化时间轴
   - 先秦思想、汉唐气象、宋元审美、明清技艺、当代表达
6. 网站功能展示
   - 注册登录、内容浏览、点赞评论、后台统计
7. 页脚
   - 素材来源说明入口
   - 课程设计说明
   - 联系信息或作者信息

#### 游客限制交互

- 点击专题页、详情页、点赞、评论、转发时，弹出登录提示弹窗。
- 弹窗文案：`请先登录后继续浏览完整内容`。
- 按钮：`去登录`、`先看看首页`。

---

### 4.2 注册页 `pages/register.html`

**访问权限：游客可访问。**

#### 页面模块

1. 左侧品牌插画区
   - 宣纸背景、卷轴卡片、文化纹样
2. 右侧注册表单
   - 用户名
   - 邮箱或手机号
   - 密码
   - 确认密码
   - 用户协议勾选
   - 注册按钮
   - 已有账号，去登录

#### 前端校验

| 字段 | 校验规则 |
|---|---|
| username | 3-16 位，允许字母、数字、下划线或中文昵称 |
| email | 符合邮箱格式，或改为手机号格式 |
| password | 至少 6 位，建议包含字母和数字 |
| confirmPassword | 必须与 password 一致 |
| agreement | 必须勾选 |

#### 状态设计

- 默认状态
- 输入错误状态
- 注册中 loading 状态
- 注册成功弹窗
- 用户名/邮箱已存在错误提示

---

### 4.3 登录页 `pages/login.html`

**访问权限：游客可访问。**

#### 页面模块

1. 登录表单卡片
   - 用户名 / 邮箱
   - 密码
   - 验证码输入框
   - 验证码图片或 Canvas 验证码区域
   - 刷新验证码按钮
   - 登录按钮
   - 去注册链接
2. 辅助信息
   - 忘记密码可仅作为静态链接，不必须实现

#### 验证码交互

- 验证码采用 4 位字符图形验证码。
- 点击验证码区域可刷新。
- 输入错误时提示：`验证码错误，请重新输入`。
- 登录成功后，根据身份跳转：
  - 普通用户：`pages/ideas.html` 或上次访问页
  - 管理员：`admin/dashboard.html`

### 4.7 无权限页 / 弹窗

由于课程要求未登录用户只能访问首页，其余子页要屏蔽，因此需要统一的权限处理。

#### 方案 A：跳转登录页

未登录访问子页时直接跳转：

```text
pages/login.html?redirect=目标页面
```

#### 方案 B：显示 403 页面

设计 `pages/no-access.html`，页面提示：

```text
你当前以游客身份访问，请登录后进入完整数字展馆。
```

推荐使用方案 A，体验更直接。

## 6. 视觉设计系统

### 6.1 色彩系统

| 用途 | 色值 | 说明 |
|---|---|---|
| 主色 Primary | `#8A1F1D` | 朱砂红，用于主按钮、重点标题、印章元素 |
| 辅色 Gold | `#C9A45A` | 鎏金色，用于边框、图标、强调线 |
| 背景 Paper | `#F6F0E5` | 宣纸米白，用于页面背景 |
| 深色 Ink | `#1F2522` | 墨色，用于正文和导航 |
| 青绿 Jade | `#2F6F5E` | 青绿山水色，用于标签和次级按钮 |
| 浅灰 Mist | `#E9E1D1` | 分割线、卡片边框 |
| 错误 Red | `#B42318` | 表单错误提示 |
| 成功 Green | `#2E7D32` | 成功状态 |

### 6.2 字体建议

```css
font-family: "Noto Serif SC", "Source Han Serif SC", "Songti SC", serif;
body-font: "Noto Sans SC", "Microsoft YaHei", sans-serif;
```

- 标题：宋体/思源宋体风格，突出传统文化感。
- 正文：无衬线字体，保证网页阅读体验。
- 数字与后台表格：使用系统无衬线字体，提高可读性。

### 6.3 图形元素

1. 宣纸纹理背景，不要过重。
2. 山水线稿作为大面积淡背景。
3. 朱砂印章作为 Logo 或装饰点。
4. 卡片角落可使用传统回纹、云纹、窗棂纹样。
5. 图标采用线性风格，避免过度卡通化。

### 6.4 布局规范

| 屏幕 | 断点 | 布局 |
|---|---|---|
| Mobile | `< 768px` | 单列布局，底部或折叠菜单 |
| Tablet | `768px - 1199px` | 双列卡片，顶部导航压缩 |
| Desktop | `>= 1200px` | 12 栅格，最大宽度 1200px |

通用容器：

```css
.container {
  max-width: 1200px;
  margin: 0 auto;
  padding: 0 24px;
}
```

## 8. 前端交互流程

### 8.1 游客访问流程

```text
进入 index.html
    ↓
浏览首页公开内容
    ↓
点击专题 / 详情 / 点赞 / 评论 / 转发
    ↓
判断未登录
    ↓
弹出登录提示或跳转 login.html
```

### 8.2 注册登录流程

```text
register.html 填写信息
    ↓
前端校验
    ↓
提交到后端注册接口
    ↓
数据库新增用户
    ↓
跳转 login.html
    ↓
输入账号、密码、验证码
    ↓
后端验证用户信息和验证码
    ↓
登录成功，写入 session / token
    ↓
跳转文化专题页
```

### 8.3 内容互动流程

```text
用户进入 detail.html
    ↓
加载文章内容和互动数据
    ↓
点击点赞 / 评论 / 转发
    ↓
调用后端接口
    ↓
数据库更新 interaction / comment 表
    ↓
前端局部刷新互动数量
```

### 8.4 后台统计流程

```text
管理员登录
    ↓
进入 admin/dashboard.html
    ↓
请求统计接口
    ↓
展示文章、用户、点赞、评论、转发数据
    ↓
可进入评论管理或互动数据页面查看明细
```

## 9. 前后端接口占位设计

前端 Agent 设计页面时需要预留以下接口调用位置。

### 9.1 用户接口

```text
POST /api/auth/register
POST /api/auth/login
GET  /api/auth/captcha
POST /api/auth/logout
GET  /api/user/profile
```

### 9.2 内容接口

```text
GET /api/categories
GET /api/articles?category=ideas
GET /api/articles/:id
```

### 9.3 互动接口


## 11. 响应式设计要求

### 11.1 桌面端

- Header 横向导航。
- Hero 左文右图或居中大视觉。
- 专题卡片每行 3 个。
- 后台使用侧边栏 + 主内容区。

### 11.2 平板端

- Header 保留 Logo，导航可压缩。
- 专题卡片每行 2 个。
- 后台侧边栏可缩窄为图标栏。

### 11.3 移动端

- Header 使用汉堡菜单。
- Hero 标题缩小，按钮纵向排列。
- 专题卡片单列。
- 文章详情页目录隐藏或折叠。
- 评论框固定在内容下方，不使用复杂浮层。

## 12. 素材来源规范

每个内容卡片和详情页都需要保留素材来源字段。

推荐在 `docs/sources.md` 中统一记录：

```markdown
# 素材来源记录

| 页面 | 素材名称 | 类型 | 来源网站 | URL | 使用位置 |
|---|---|---|---|---|---|
| index.html | 山水背景图 | 图片 | xxx | xxx | 首页 Hero |
| ideas.html | 民为邦本文案 | 文本 | xxx | xxx | 思想理念页 |
```

前端页面中建议在详情页底部显示：

```text
素材来源：资料整理自……，图片来源：……
```

## 14. 验收清单

### 14.1 课程硬性要求

- [ ] 网站主题围绕中华优秀传统文化。
- [ ] 首页文件名为 `index.html`。
- [ ] 至少包含三级页面。
- [ ] 目录结构使用非中文名称。
- [ ] 页面风格统一，布局美观，色彩协调。
- [ ] 实现响应式适配。
- [ ] 注册功能连接数据库。
- [ ] 登录功能包含验证码。

- [ ] 登录用户可以点赞、评论、转发。

## 15. 最终建议

前端页面不要只做静态展示，应在视觉设计中明确体现“游客限制、登录验证码、注册、点赞、评论、转发、后台统计”这些课程评分点。Stitch 生成页面时，应优先生成完整页面框架和统一设计系统，后续再由前端 Agent 将交互和接口接入实际代码。
